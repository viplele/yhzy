What is DoH (DNS over HTTPS)?

DNS over HTTPS (DoH) is a protocol that encrypts and authenticates the communication between DNS clients and servers using the HTTPS protocol. The goal of DoH is to enhance DNS privacy and security by encrypting query and response payloads and providing authentication of the DNS server identity. This protects DNS queries from eavesdropping and tampering on the network, client, or server side.

With DoH, DNS queries are sent over an HTTPS connection, meaning they are encoded in the same format as web requests and are subject to the same encryption and authentication features as web traffic. This makes it more difficult for attackers to intercept or tamper with DNS queries or associate them with users' browsing activities.

Qu'est-ce que DoH (DNS over HTTPS) ?

DNS over HTTPS (DoH) est un protocole qui chiffre et authentifie la communication entre les clients et les serveurs DNS en utilisant le protocole HTTPS. L'objectif de DoH est d'améliorer la confidentialité et la sécurité de DNS en chiffrant les charges utiles des requêtes et des réponses, et en fournissant une authentification de l'identité du serveur DNS. Cela protège les requêtes DNS contre l'écoute et la falsification sur le réseau, le côté client ou serveur.

Avec DoH, les requêtes DNS sont envoyées via une connexion HTTPS, ce qui signifie qu'elles sont codées dans le même format que les requêtes web et sont soumises aux mêmes fonctionnalités de cryptage et d'authentification que le trafic web. Cela rend plus difficile pour les attaquants d'intercepter ou de falsifier les requêtes DNS ou de les associer aux activités de navigation des utilisateurs.

什么是 DoH (DNS over HTTPS)？

DNS over HTTPS （DoH） 是一种使用 HTTPS（HTTP over Transport Layer Security）协议对 DNS 客户端和服务器之间的通信进行加密和身份验证的协议。DoH 的目标是通过加密查询和响应有效负载以及提供 DNS 服务器身份的身份验证来提高 DNS 的隐私和安全性。这样就可以保护 DNS 查询在网络和客户端或服务器端免受窃听和篡改。

使用 DoH，DNS 查询通过 HTTPS 连接发送，这意味着它们以与 Web 请求相同的格式进行编码，并受到与 Web 流量相同的加密和身份验证功能的约束。这会使攻击者更难拦截或篡改 DNS 查询，或将查询与用户的浏览活动相关联。
